## 待诊断的需求

$ARGUMENTS

## 分析流程（按顺序执行，不要跳步）

1. **四大误判筛查**：逐条对照，这个需求是否踩坑？
   - 混淆第一性与第二性需求？
   - 把真实场景的摩擦当成抽象场景的价值？
   - 忽略机会成本与多维竞争？
   - 错判需求的深度与广度？
   每一条给出"踩 / 没踩 / 需要更多信息"的判断，踩了就解释为什么。

2. **三问法演练**：如果今天要做用户调研，列出具体的问题。**禁止**假设性提问（"如果我做了 X 你会用吗"），**只能**问过去和现在的真实行为。
   - 问法 1：回忆真实行为（验证痛点存在性）
   - 问法 2：评估机会成本（明确真正竞品）
   - 问法 3：测试支付意愿 / 克服摩擦的动力

3. **检查清单过一遍**：6 项每项给出"✅ / ❌ / ⚠️ 需验证"。

4. **判决**：真需求 / 伪需求 / 证据不足需验证。给出**下一步动作**（该做什么用户调研 / 该砍什么 / 该怎么小成本验证）。

## 输出原则

- 不要泛泛而谈，每条都要贴着用户的具体需求说话
- 举反例优先于讲概念
- 如果用户信息不够，直接指出"我需要知道 X 才能判断"，不要硬给结论
