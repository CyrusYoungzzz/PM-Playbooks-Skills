## 诊断流程

### Step 1 · 四对齐扫描

如果用户对你的产品没有惊喜感（没有传播欲），一定是以下某个或某几个对齐发生了。逐条判断：

| 对齐类型 | 症状 | 是否命中 | 证据 |
|---------|------|---------|------|
| **功能对齐** | "果然是高配旗舰" | 是 / 否 | ... |
| **价格对齐** | "果然是这个价位" | 是 / 否 | ... |
| **体验对齐** | "果然是这样操作的" | 是 / 否 | ... |
| **叙事对齐** | "果然是 XX 风格" | 是 / 否 | ... |

命中一个以上就诊断成立。

### Step 2 · 贬值信号扫描（框架 7）

判断是不是"超预期已经贬值"了：
- [ ] 用户还会主动发朋友圈/小红书/微博吗？（不发 = 贬值信号 1）
- [ ] 竞品是否开始致敬你的手法？（是 = 贬值信号 2）
- [ ] 你的团队是否已经把"超预期"写进 PRD？（是 = 贬值信号 3）

### Step 3 · 定位破局维度

**核心原则**：先确认用户的"果然如此"落在哪个维度上，然后就在那个维度上制造违背。

结合 Step 1 的对齐结果，给出 3 种破局路径（参考框架 7 的三策略）：

- **升层**：如果目前是功能对齐，能否升到体验超预期？如果是体验对齐，能否升到情感超预期？
- **降维打击**：所有竞品在卷哪个维度？你能否跳到另一个维度？（例：全行业卷摄像头 → 苹果做灵动岛）
- **反预期**：用户预期你「更便宜」时做「更贵但值得」；预期你「更多功能」时做「更少但更精」

每条路径给出**具体动作**，不要停在概念上。

### Step 4 · 诊断结论

```
## 诊断

主要对齐：...（功能 / 价格 / 体验 / 叙事）
贬值阶段：未贬值 / 早期信号 / 已深度贬值
根因：...

## 破局建议（按优先级）

1. ...（升层 / 降维 / 反预期 中的一种）
2. ...
3. ...

## 下一步动作

- 本周内可以做的小实验：...
- 下个版本需要的改动：...
- 需要团队共识的决策：...
```

## 原则

- 不要把"四对齐"全判成"是"就完事了，必须定位出**最关键**的那一个
- 破局建议必须避开同样的陷阱（如果用户已经功能对齐了，别再建议"多加几个功能"）
