## 设计流程

### Step 1 · 锚点诊断（框架 2）

用户在接触这个产品前，脑子里会有什么锚点？四个来源逐一回答：
- 品类锚点：用户对这个品类的整体认知是什么？
- 价格锚点：你的定价信号会触发什么记忆模板？
- 外观锚点：第一眼会暗示什么档次？
- 社交锚点：用户在看到你之前刷到过什么？

**关键判断**：哪几个锚点应该**主动压低**？哪几个维度上你打算**掀翻**？

### Step 2 · 选择层级

对照四层，选择适合的打法（可叠加）：

- **第一层 · 剧情反转**：单点功能溢出。是否能藏一个"用户自己发现"的强超预期？
- **第二层 · 叠加反转**：跨维度连续小反转。画完整用户旅程，每个节点放一个**不同维度**的超预期。维度散度是关键。
- **第三层 · 抽象式体验**：无目的的愉悦。有没有一个"没道理但让用户觉得被在意"的细节？（参考 AirPods 弹窗）
- **第四层 · 回声效应**：跨产品矩阵。本品和品牌已有产品如何"变奏不变调"？

### Step 3 · 旅程节点设计（核心产出）

输出一张「超预期递进表」：

| 节点 | 用户此刻预期 | 你的反转 | 反转维度 | 强度 | 是否自发现 |
|------|------------|---------|---------|------|----------|
| 广告/宣传 | ... | ... | 价格 | 弱 | 否 |
| 开箱 | ... | ... | 工艺 | 中 | 是 |
| 首次使用 | ... | ... | 效率 | 中 | 是 |
| 深度使用一周 | ... | ... | 情感 | 强 | 是 |

强度递增、维度互不重复、节点间隔 < 用户消化时间。

### Step 4 · 反模式检查

逐项排查（参考`~/.claude/skills/pm-playbooks/references/lichuangqi-baopin.md`（或项目内 `.claude/skills/pm-playbooks/references/lichuangqi-baopin.md`） 反模式速查表）：
- [ ] 所有超预期点是否都堆在同一个维度（多半是"配置"）？→ 重新散点
- [ ] 是否在发布前把所有功能都讲透了？→ 留给用户自己发现
- [ ] 抽象式体验有没有做成弹窗彩蛋？→ 藏起来
- [ ] 是否在用"贵材质"假装"在意"？→ 在意 ≠ 花钱

## 输出原则

- 节点要具体到"用户此刻在做什么动作"，不要写"用户感受到惊喜"这种空话
- 反转维度必须明确标注（交互 / 操控 / 场景 / 品类 / 工艺 / 效率 / 情感 / 价格 / 速度 …），同维度不能连续出现两次
- 至少有一个节点必须是"用户自己发现"，而不是产品主动告知
