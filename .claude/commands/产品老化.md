## 诊断流程

### Step 1 · 贬值信号扫描

逐条检查（每条需要具体证据）：

| 信号 | 判断 | 证据 |
|-----|------|------|
| 用户不再发朋友圈/社媒 | 是 / 否 | ... |
| 竞品开始"致敬"你的手法 | 是 / 否 | ... |
| 团队把"超预期"写进 PRD | 是 / 否 | ... |

**命中 2 条以上 → 深度贬值**。

### Step 2 · 锚点漂移定位

李创奇的核心洞察：你第一次成功后，**用户会用你的超预期作为新的基线**。

- 第一代的锚点是什么？（例：小米手环第一代，锚点是"白牌几十块"）
- 现在的锚点是什么？（例：现在变成"小米=便宜好用"）
- "便宜"→"果然便宜"的漂移是否已经发生？
  - "果然便宜"比"便宜"更危险——它已经变成预期本身，你再便宜也只是达标

### Step 3 · 选策略

根据贬值深度，选择应对（可组合）：

**策略 A · 升层**
- 当前用的是哪一层超预期？（1 剧情反转 / 2 叠加反转 / 3 抽象式体验 / 4 回声效应）
- 能否升到下一层？
- 注意：升层是改变**类型**，不是简单加功能
  - 功能超预期 → 体验超预期：例如从"做更多"到"做得让用户被在意"
  - 体验超预期 → 情感超预期：例如抽象式体验 / 品牌认同

**策略 B · 降维打击**
- 这个赛道所有人都在卷哪个维度？（看竞品最近 3 次迭代重点）
- 有没有一个维度所有人都忽略了？
- 案例：所有手机厂商卷摄像头 → 苹果做灵动岛（全新交互方式）
- 你能在哪个新维度上发力？

**策略 C · 反预期**
- 用户现在预期你"更 X"（便宜 / 多功能 / 轻薄 / ...）
- 反预期做法：做"不 X 但值得"的
  - 用户预期你便宜 → 做"不便宜但值得"的（打破第四面墙）
  - 用户预期你多功能 → 做"更少但更精"的
- 这在喜剧里叫"反包袱"

```

### 锚点漂移

### 推荐策略（按优先级）

   - 具体动作：...
   - 验证方式：...

2. **备选策略：...**
   - ...

### 做 or 不做清单
继续做：
- ...
立刻停止：
- ...
（停止的那些往往是已经变成"果然如此"的部分）

### 禁区警告
不要做的事（会加速贬值）：
- 继续在已经被用户视为基线的维度上加码
- 把"超预期"流程化写进 PRD
- 通过弹窗/Toast 主动宣告"你发现了彩蛋"
```

## 原则

- 如果已经深度贬值还在用第一次的手法，就是在加速死亡
- 变奏不变调：品牌的"核心精神"不能丢，但具体手法必须换
- 如果三策略都不适用（小众赛道、没竞品可借鉴），考虑回到 `/品类立项` 重新判断这个品类的拐点是否还在
