先读取 `~/.claude/references/lichuangqi-baopin.md`（或项目内 `.claude/references/lichuangqi-baopin.md`），重点查阅「框架 1 · 第四层 · 回声效应」。然后为下面的品牌矩阵设计回声节奏：

## 当前品牌矩阵 / 计划新品

$ARGUMENTS

## 设计流程

### Step 1 · 已有产品的"超预期手法"梳理

把品牌已有的爆品列出来，标注每个产品当年用什么手法超预期：

| 产品 | 上市时间 | 超预期手法 | 反转维度 |
|------|---------|----------|---------|
| 例：小米手环 | 2014 | 79 元 + 30 天续航 + 通知 + 解锁 | 价格 + 功能 |
| 例：移动电源 | 2013 | 69 元 10400mAh + 做工 | 价格 + 品质 |
| 例：空气净化器 | 2014 | 899 + 设计感（客厅不丢人） | 价格 + 设计 |
| ... | ... | ... | ... |

### Step 2 · 提炼"不变调"

回声效应的核心是「**变奏不变调**」：

- 这个品牌的"不变调"是什么？（例：小米 = 价格 + 体验双超预期）
- 这是从用户视角能感受到的核心精神，不是 PPT 里的品牌主张

### Step 3 · 检查"变奏"是否充分

逐对检查相邻产品：

- 它们的反转维度是否相同？（如果都用"价格+功能"，用户的防御模型已经追上）
- 维度散度足够吗？

### Step 4 · 节奏诊断

- 上次回声产品距今多久？
- 节奏是否合理？
  - 太久 → 用户会遗忘
  - 太频繁 → 用户会疲倦
  - 经验值：1 年 2-3 个回声产品

### Step 5 · 回声疲劳预警

- 用户对你的预期是否已经从"X"演化到"果然 X"？（例：便宜 → 果然便宜）
- 如果是，必须打破第四面墙：在用户以为你只会做 X 时，突然做一个**不 X 但值得**的产品

### Step 6 · 新品定位建议

为计划新品给出回声设计：

```
## 新品建议

### 不变调
继承的核心精神：...

### 变奏维度
本品超预期手法：（不能与最近 2 个回声产品相同）
反转维度：（明确标注：交互/操控/场景/品类/工艺/效率/情感/价格/速度/设计 …）

### 节奏匹配度
距上一回声：X 个月（合理 / 太快 / 太慢）

### 回声疲劳预警
当前用户对品牌的"果然如此"是：...
本品是否打破这个预期？...

### 第一次超预期的"品牌印记"
（如果是新品类首发）让用户记住的不只是"这个产品好"，而是"这个品牌有意思"——印记是：...
```

## 原则

- 不变调比变奏更重要——如果"核心精神"没传递清楚，再多产品也建不起回声
- 变奏维度的选择要主动避开品牌的"高频维度"（小米的高频是"价格"，所以新品的反转维度要是只有价格就单调）
- 如果品牌已经进入"果然 X"阶段，回声效应不再起效，需要切到 `/产品老化` 走反预期策略
