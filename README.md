# PM-Playbooks.skill

> *「不是所有需求都值得做，也不是每个功能都能成爆品。」*

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Claude Code](https://img.shields.io/badge/Claude%20Code-Slash%20Commands-blueviolet)](https://claude.ai/code)

<br>

**把产品大牛的方法论拆成可触发的 `/command`，按场景调用，避免方法论散落在博客和书里没人看。**

<br>

基于课代表立正、李创奇等产品人的公开文章 / 演讲 / 采访等素材蒸馏整理。<br>
不是复读语录，是用他们看产品的方式帮你诊断你手上的需求。

<br>

**主要面向 AI 硬件产品**（AI 眼镜 / 可穿戴 / 智能家居等）。

> **课代表立正** · B 站百万粉丝产品 UP 主，擅长用"真伪需求"方法论拆解产品，是中文互联网产品思维启蒙号之一。<br>
> **李创奇** · 前小米可穿戴创新品类负责人，主导小米电视 / 米家投影 / 巨能写 / 小米 AI 眼镜等爆品。

## 效果示例

```
用户      ❯ /真伪需求分析 我想给陪聊 App 加一个"AI 自动给好友送语音祝福"
            功能，用户可以一键定时给联系人发周年纪念祝福

PM        ❯ 先按"四大误判"过一遍：

            ❌ 第一性 vs 第二性混淆
              用户的第一性需求是"被记住"，不是"自动发祝福"。
              真把祝福外包给 AI，反而让收到的人觉得"原来你没想我"。

            ❌ 抽象场景的伪价值
              "周年纪念"听起来很美，但用户上次主动给朋友发祝福
              是什么时候？如果本来就不发，加个 AI 不会变成发。

            ⚠️ 机会成本未评估
              真要解决"维系关系"，对手是微信、是朋友圈，不是
              另一个 AI 功能。

            三问法（只问真实行为，不问假设）：
            1. "上次主动给朋友发节日祝福是什么时候？发了什么？"
            2. "你现在用什么方式记朋友的纪念日？"
            3. "如果让你每月花 9 块钱让 AI 帮你记，你会订阅吗？"

            判决：伪需求倾向。下一步——找 5 个"自称重视关系"
            的用户，看他们手机里有没有任何"关系维系"工具的使用
            痕迹。没有就直接砍。
```

```
用户      ❯ /爆品立项 做一款主打"零延迟"的开源蓝牙耳机，
            定价 199，瞄准学生群体

PM        ❯ 用李创奇"长板理论 + 立项三问"过一遍：

            立项三问：
            1. 这个品类还有没有"未被满足的预期"？
               → 199 价位的延迟体验已经被红米 Buds、漫步者
                 拉到 80ms 内。你说的"零延迟"如果只是 60ms，
                 用户感知不到，不构成预期违背。

            2. 你的长板能不能撑起一句话卖点？
               → "开源"对学生群体不是卖点，是噪音。
                 学生买耳机看的是颜值 + 续航 + 通话。

            3. 销量反推 ROI：
               → 199 × 50 万台 = 1 亿。蓝牙耳机品类年销 3 亿
                 副，你要拿 0.16% 份额。可行但需要一个真实
                 的预期违背点支撑。

            建议：要么把"零延迟"做到 30ms 以内（量化的预期
            违背），要么把"开源"换成对学生有感的钩子（如
            "丢一只补一只"）。否则不立。
```

```
用户      ❯ /预期诊断 我们的 PWA 工具上线 3 个月，DAU 稳定
            但分享率从 8% 跌到 2%，用户访谈反馈"挺好用的"

PM        ❯ "挺好用的" = "果然如此"信号，典型的预期对齐贬值。

            果然如此四对齐自检：
            ✅ 功能对齐：你做的就是用户预期的，没惊喜
            ✅ 体验对齐：交互符合直觉，但也仅此而已
            ✅ 价格对齐：免费，没"占便宜"感
            ✅ 品牌对齐：用户已把你归类为"工具"，停止讨论

            分享率从 8% 跌到 2% 的本质：用户不再有"想说给
            朋友听"的冲动。第一次用是新鲜，三个月后变默认。

            下一步——必须制造一次"预期违背"：
            • 升层：在工具里加一个反工具的玩法（如随机彩蛋）
            • 降维：把核心功能做成 30 秒可分享的轻量结果页
            • 反预期：搞一次"工具自黑"运营事件

            不要再做"再好用一点"的迭代，对分享率没用。
```

这不是空泛的方法论复读。每个命令都按蒸馏出来的真实框架——「四大误判 + 三问法」「立项三问 + 销量反推」「果然如此四对齐」——逐项诊断，最后给出**可执行的下一步**而不是"加油你行的"。

## 安装

本仓库基于 **Claude Code 的 [Slash Commands](https://docs.claude.com/en/docs/claude-code/slash-commands)** 实现。

### 一键安装（推荐）

```bash
curl -fsSL https://raw.githubusercontent.com/CyrusYoungzzz/PM-Playbooks-Skills/main/install.sh | bash
```

默认安装到 `~/.claude/`（所有项目都能用）。想只装到当前项目：

```bash
curl -fsSL https://raw.githubusercontent.com/CyrusYoungzzz/PM-Playbooks-Skills/main/install.sh | bash -s -- --project
```

### 验证

进入任一项目，启动 Claude Code，输入 `/` 应该能看到 `/真伪需求分析` 等 7 条命令。试一下：

```
/真伪需求分析 我想给陪聊 App 加一个一键定时祝福功能
```

### 手动安装（不信任 curl | bash 的看这里）

```bash
git clone https://github.com/CyrusYoungzzz/PM-Playbooks-Skills.git
cp -R PM-Playbooks-Skills/.claude/commands/* ~/.claude/commands/
cp -R PM-Playbooks-Skills/references ~/.claude/
```

### 卸载

```bash
rm -rf ~/.claude/commands/{真伪需求分析,爆品立项,超预期设计,预期诊断,品类立项,产品老化,回声效应}.md
rm -rf ~/.claude/references
```

### 其他工具

见上方「在哪里可以用」表格。Cursor / VSCode 等可手动迁移命令内容到 Rules / Prompt 模板使用，但失去 `/command` 触发的便利

## 使用

每个命令对应一个具体的产品场景。直接在 AI 编程工具里输入：

```
> /真伪需求分析 [你的需求描述]
> /爆品立项 [产品想法 + 定价 + 目标人群]
> /超预期设计 [要设计的产品 / 功能]
> /预期诊断 [用户反馈 / 数据现象]
> /品类立项 [新品类 + 时机判断]
> /产品老化 [迭代到第 N 代变平庸的产品]
> /回声效应 [品牌矩阵 / 新品节奏规划]
```

命令会自动读取对应的 reference 文档，按方法论的固定流程逐项分析，输出"判决 + 下一步动作"。

## 蒸馏了什么

不是学院派 MBA 教材，是从一线产品人的公开素材里提取的真实决策框架：

| 来源 | 心智模型 | 一句话 |
|------|---------|--------|
| **课代表立正**（B 站 UP） | 真伪需求四大误判 | 第一性 vs 第二性、抽象场景陷阱、机会成本盲区、需求深度广度错判 |
| **课代表立正** | 三问法 | 只问真实行为，不问假设——"上次什么时候做的？怎么做的？花了多少钱？" |
| **李创奇** | 预期违背四层 | 功能 / 体验 / 价格 / 品牌——任何一层制造惊喜都能引发传播 |
| **李创奇** | 长板理论 | 别补短板，把一根长板捅穿天花板，用户记得住的永远是那一根 |
| **李创奇** | 爆品三方法论 | 销量反推、预期锚点四来源、果然如此四对齐 |
| **李创奇** | 回声效应 | 新品要借上一次的预期势能——变奏不变调，矩阵节奏比单点爆破更重要 |
| **李创奇** | 立项三问 | 未被满足的预期？长板能不能成卖点？销量反推 ROI 算得过吗？ |

**为什么是这两位？**
- 课代表立正擅长在需求阶段做减法——帮你识别"看起来很美但没人会用"的伪需求
- 李创奇擅长在立项 + 体验阶段做加法——帮你把"还不错"做到"忍不住分享"

两者搭配，覆盖产品决策从"要不要做"到"怎么做出爆款"的完整链路。

## 诚实边界

**这个 Skill 能做的：**
- 用真实的产品决策框架诊断你手上的需求 / 立项 / 体验问题
- 强制走完"判决 + 下一步动作"的流程，避免变成空话
- 把方法论从"读了就忘"变成"按场景触发"

**做不到的：**

| 维度 | 说明 |
|------|------|
| 替代用户调研 | 命令会告诉你"该问哪三个问题"，但真实数据要你自己去拿 |
| 行业 know-how | 涉及具体行业（医疗、金融、硬件供应链）的技术细节不在覆盖范围 |
| 替代本人决策 | 蒸馏的是公开方法论，最终决策的责任在你 |
| 实时跟踪最新动态 | 素材有截止日期（约 2026-04），最新言论可能未收录 |
| 一键出爆品 | 方法论降低犯傻概率，不保证成功 |

**一个不告诉你局限在哪的 Skill，不值得信任。**

## 仓库结构

```
PM-Playbooks-Skills/
├── SKILL.md                          # Skill 入口 / 命令清单
├── README.md                         # 本文件
├── CONTRIBUTING.md                   # 扩展指南（怎么加新方法论）
├── index.md                          # 原始素材索引
├── .claude/commands/                 # 命令路由（短，只规定流程）
│   ├── 真伪需求分析.md
│   ├── 爆品立项.md
│   ├── 超预期设计.md
│   ├── 预期诊断.md
│   ├── 品类立项.md
│   ├── 产品老化.md
│   └── 回声效应.md
├── references/                       # 方法论沉淀（命令按需加载）
│   ├── kedaibiao-real-vs-fake.md
│   └── lichuangqi-baopin.md
├── evals/                            # 测试用例（改方法论后跑一遍防退化）
│   ├── evals.json                    #   7 条命令各 1 个 prompt + expectations
│   └── README.md
├── kedaibiao-raw/                    # 课代表立正原始素材
└── lichuangqi-raw/                   # 李创奇原始素材
    ├── interviews/                   #   公开采访 / 演讲
    └── zhihu/                        #   知乎文章 / 回答 / 想法
```

设计原则：**命令是路由不是知识**。命令文件本身要短，方法论放 `references/`，命令第一步加载它——避免一次性把所有方法论塞进上下文。详见 [CONTRIBUTING.md](./CONTRIBUTING.md)。

## 怎么扩

新增一个产品方法论时：

1. 把原始素材放到 `<人名>-raw/`（爬下来的文章 / 视频字幕 / 书摘）
2. 蒸馏出一个 `references/<人名>-<主题>.md`（按方法论的天然结构组织）
3. 为每个使用场景写一个 `.claude/commands/<场景名>.md`
4. 在 [SKILL.md](./SKILL.md) 的「当前收录」「命令清单」表格里追加一行

完整规范见 [CONTRIBUTING.md](./CONTRIBUTING.md)。
