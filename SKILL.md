# Product 方法论

一套面向产品决策的命令式工具集。把不同产品大牛的方法论拆成可触发的 `/command`，按场景调用，避免方法论散落在博客和书里没人看。

## 典型 Workflow（命令怎么串）

单个命令只能解决单一问题。真正的产品决策往往需要**多个命令串联**才能落地。以下是四种典型场景的推荐链路——**任一步打回，就止步、回去修证据，不要硬往下走**。

### Workflow 1 · 新想法 → 立项落地

场景：有一个新产品 / 新功能想法，想判断要不要做、怎么做成爆款。

```
/真伪需求分析  →  判决「真需求」？
                       │
                       ├─ 伪需求  →  止步，按输出的调研清单先去找证据
                       │
                       └─ 真需求  →  /爆品立项  →  评分「高」？
                                              │
                                              ├─ 中/低 → 止步或退回修长板
                                              │
                                              └─ 高   →  /超预期设计  →  落地方案
```

**关键节点**：
- 伪需求阶段绝对不要进 `/爆品立项`——好长板救不了假场景
- `/爆品立项` 评"中"时，常见错误是硬凑第 2、3 个长板。正确做法是回到 `/真伪需求分析` 看是不是需求本身就不够硬
- `/超预期设计` 产出的递进表应该指导后续 PRD 写作

### Workflow 2 · 老产品救火

场景：存量产品数据下滑（分享率 / 留存 / 销量）、用户反馈"还行但没什么新鲜感"。

```
/预期诊断  →  贬值阶段「未贬值 / 早期 / 深度」？
                │
                ├─ 未贬值  →  可能是渠道/运营问题，本 skill 帮不上
                │
                ├─ 早期    →  按诊断输出的单一策略执行即可
                │
                └─ 深度    →  /产品老化  →  产出升层/降维/反预期组合拳
                                      │
                                      └─ 如果三策略都不适用 → /品类立项
                                         （重新判断这个品类的拐点是否还在）
```

**关键节点**：
- `/预期诊断` 是"定位"，`/产品老化` 是"破局"。别跳过定位直接找破局
- 深度贬值的破局不是"再做一代更好的"，而是必须**换维度**

### Workflow 3 · 新品类战略入局

场景：要评估进入一个全新硬件品类（AI 眼镜、机器人、可穿戴新形态等）。

```
/品类立项  →  判断「值得做 / 慎重 / 不该做」？
              │
              ├─ 不该做  →  止步，按红灯警告反思动机
              │
              ├─ 慎重    →  按销量反推的"安全数字"重算 ROI，达不到就止步
              │
              └─ 值得做  →  /爆品立项（第一代单品）
                          →  /超预期设计（首发体验）
                          →  /回声效应（规划 2-3 年内的矩阵节奏）
```

**关键节点**：
- `/品类立项` 的"谁的机会"判断决定后续策略：创业公司硬刚大厂赛道必须在软件 / OS / 生态层建立独立壁垒
- 品类入局的第一代目标**不是利润**，要在 `/爆品立项` 里主动调整期望

### Workflow 4 · 品牌矩阵规划

场景：已有多款产品的品牌，要规划年度新品节奏 / 子品牌矩阵。

```
/回声效应  →  识别「不变调」+ 检查「变奏维度散度」
              │
              ├─ 变奏充足  →  按输出的新品定位建议执行
              │
              ├─ 变奏不足  →  对具体新品跑 /爆品立项，强制换维度
              │
              └─ 进入"果然 X"  →  对核心产品跑 /产品老化，走反预期策略
```

**关键节点**：
- 回声效应的前提是"品牌还在被期待"——如果用户已经对你"果然如此"，回声会变成自我重复
- 矩阵节奏经验值：**1 年 2-3 个回声产品**。太快疲劳，太慢遗忘

## 反模式：单命令就交差

常见错误：

- ❌ 跳过 `/真伪需求分析` 直接跑 `/爆品立项`——在伪需求上凑长板只会做出"好但没人用"的产品
- ❌ 对深度贬值的老产品只跑 `/预期诊断` 不跑 `/产品老化`——诊断出问题没给破局路径等于没做
- ❌ 对全新品类跳过 `/品类立项` 直接 `/爆品立项`——品类拐点没到时，爆品方法论救不了你
- ❌ 把 7 条命令当成选择题只用最熟的那一条——每条命令的适用场景不重叠，选错命令比没用命令更糟

## 调用命令时的心态

- **证据不足时不要硬给结论**。命令输出"需要知道 X 才能判断"比"我觉得可以做"更有用
- **当命令的输出是"打回"时，认真对待**。PM 最贵的不是想法，是没被真伪筛掉的想法消耗的资源
- **不要把命令输出当 PRD**。命令给的是"判决 + 方向"，落地仍需要你自己写可执行计划
